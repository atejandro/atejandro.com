Start a blog using Hexo and GitHub Pages

2015-02-04

Recently, I’ve been trying to ‘explore new horizons’ learning new programming languages and frameworks. I’m a Java guy and I really like it! But I know that you can get stuck or ‘comfortable’ with your daily living. Therefore, I started investigating about Node.js and its frameworks. To my surprise, I stumbled upon Hexo and its beautiful way to easily deploy your posts.

This tutorial will help you with the basics for installing, generating, previewing and deploying your blog to Github Pages.

Hexo is a really powerful open source blog framework powered by Node.js. It is installed in your system and with simple commands, it generates hundreds of static files in a blink of an eye ( I would say that it is faster than Octopress). Some of the most awesome features that Hexo has are:

  • Publish, edit and preview your post in private.
  • Easy deployment to Github and Heroku
  • Manage your posts with categories, tags, assets & menus.

Ok, enough with the chit chat. Let’s get started

Getting started with Hexo

The first step is to install Hexo in your system:

You need to know that Hexo solely depends on Node.js, so it has to be previously intalled in your system (here is an easy way to get started with Node.js in case you don’t know how to :) ). Now you have your first blog entry. Pretty easy huh?

You’ll notice that your first post is empty. To add content, edit the generated file under source/_posts/first-post.md and follow the Markdown Syntax to enrich the text properly.

Deploying the blog to Github Pages

In order to deploy your blog, you’ll need to have a github account and create a repo as yourdomain.github.io. This naming convention, lets Github know that the contents of the repo might work as pages. As soon as the repository detects an index.html file, in a matter of minutes, the site will be fully available at yourdomain.github.io.

Configuring Hexo deployment options

To link Hexo to Github Pages, you need to modify the config file (_config.yml) located in the root of the hexo project. It’s contents have a lot of configuration variables to fill in, including:

  • Site’s general information, such ass title, author, email, etc.
  • Server configuration
  • URL generation
  • Extensions for adding customized themes
  • Comment support vía Disqus username
  • Deployment configuration
  • and more…

Every section of the config file is pretty self explanatory and references Hexo’s documentation for further detail. I encourage you to take a look at it and experiment when you have the time. For now, we just need to modify the ‘Deployment’ section at the end of the file.

To deploy to github, follow the format given above. Now try executing the hexo deploy command in your terminal and wait a few seconds. Hexo framework will get all the generated files and push them to the github repo. You now have your first live blog in the web!
